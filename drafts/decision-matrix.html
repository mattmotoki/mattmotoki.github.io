Decision Matrix

Matt Motoki, Sun 26 November 2017, Tue 28 November 2017, Misc

misc

If you are like me then you have a million things that you want to do but no clear order in which to do them. In this post, I will introduce the Decision Matrix which is a tool that breaks down a persons utility funciton into a linear combination of features. I relate the decision matrix to Utility theory and show how we formulate the problem of finding the optimal schedule as a linear program. I then show how I use it to prioritize my day.

We seek to maximize the expected utility of our schedule subject to the constraint that all tasks must be done on time (this constraint can be relaxed by introducing a penalty for late tasks).

Every action we take has a consequece, it has a benefit, it takes some time and sometimes it has a deadline associated with. When we are faced with the problem of choosing which action to take

Utility theory says that every rational being can be reduced to an agent who is maximizing the expected value of some utility function. The decision matrix aims to break an agent's utility into a linear combination of variables. We provide a greedy approach to scheduling task. The optimal schedule can be found via linear programming.
